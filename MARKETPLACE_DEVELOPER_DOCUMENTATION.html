1. What this project is

This is an Amazon-style marketplace web app with a product catalog, cart and checkout, customer accounts, seller onboarding with Stripe Connect, and an admin area for managing products, coupons, orders, and analytics.

4. System overview (developer)

4.1 Routing + layouts

The UI uses locale-prefixed routing under /[locale]/… with next-intl (see src/app/[locale]/layout.tsx and src/i18n/routing.ts). Admin and Seller areas are protected at the layout level using server-side session checks and redirects to login with callbackUrl.

4.2 Authentication + roles

Authentication is NextAuth with a credentials provider (email/password). Passwords are stored as bcrypt hashes in User.passwordHash. Sessions use the JWT strategy; the user role is copied into the JWT and then into session.user.role (see src/auth.ts).

4.3 Data layer

Prisma models cover users/roles, sellers, products/categories/images, carts (guest + user), orders + order lines, reviews, wishlist, coupons/redemptions, subscriptions/entitlements, and StripeEvent for webhook idempotency (see prisma/schema.prisma).

4.4 Payments

Stripe is used for one-time Checkout sessions and subscription Checkout sessions. A webhook handler finalizes paid checkouts, decrements stock, splits a single checkout into per-seller orders, records coupon redemption, clears cart, and syncs subscriptions + entitlements.

7. API endpoints (server routes)

The app uses Next.js route handlers under src/app/api (mounted at /api).

  • Auth: POST /api/auth/register, GET/POST /api/auth/[...nextauth]
  • Cart: POST /api/cart, PATCH /api/cart, DELETE /api/cart
  • Wishlist: GET/POST/DELETE /api/wishlist
  • Reviews: GET/POST /api/reviews
  • Coupons: GET /api/coupon?code=…
  • Checkout (one-time): POST /api/checkout
  • Checkout (subscription): POST /api/checkout/subscription
  • Stripe webhooks: POST /api/webhooks/stripe
  • Seller onboarding: POST /api/seller/connect
  • Upload product image: POST /api/upload/product-image
  • Private Blob streaming: GET /api/media/blob?pathname=…
  • Account export: GET /api/account/export

8. Data model (Prisma / PostgreSQL)

Core entities:

  • Users with roles: CUSTOMER, SELLER, ADMIN
  • Sellers (shop profile + Stripe Connect account id + verification status)
  • Products, categories, product images
  • Cart + cart items (guest via sessionId, logged-in via userId)
  • Orders + order lines (orders stored per-seller; checkout can split a single paid session into multiple seller orders)
  • Reviews (one review per user+product), verifiedPurchase flag
  • Wishlist items
  • Coupons + redemptions
  • Subscriptions + subscription entitlements (key: subscription_active)
  • StripeEvent table for webhook idempotency

9. Authentication and authorization

Authentication is handled by NextAuth using a credentials provider (email + password hash). Sessions are JWT-based. Role-based access is enforced in server layouts for /admin and /seller.

10. Payments and Stripe

10.1 One-time checkout

A logged-in user creates a Stripe Checkout session for cart items. The checkout payload includes shipping, optional estimated tax (when Stripe Tax is off), and optional coupons. If the cart contains items from a single seller with Stripe Connect configured, the payment intent includes an application fee and transfer destination.

10.2 Webhooks

Stripe webhooks finalize orders, decrement stock, split orders per seller, record coupon redemption, clear cart, and sync subscription lifecycle events. Idempotency is enforced via the StripeEvent table.

11. Environment variables

Create a .env file based on .env.example. Key variables:

  • DATABASE_URL: PostgreSQL connection string
  • NEXTAUTH_SECRET, NEXTAUTH_URL
  • NEXT_PUBLIC_APP_URL: public base URL for Stripe redirects
  • STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY
  • STRIPE_ENABLE_TAX, TAX_BPS (optional estimated tax)
  • BLOB_READ_WRITE_TOKEN, BLOB_ACCESS (optional Vercel Blob)

12. Notes and constraints

  • Currency amounts are stored as integers (cents) in the database and Stripe line items.
  • Cart supports guest sessions via an HTTP-only cookie.
  • Reviews are limited to one per user per product (upsert).
  • Webhook handler uses a StripeEvent table for idempotency.
